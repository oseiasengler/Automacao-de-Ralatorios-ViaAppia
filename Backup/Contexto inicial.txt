O coração desse app é o modelo de dados, e ele tem uma particularidade que define toda a arquitetura: cada dispositivo de drenagem tem atributos diferentes. Um tubo tem diâmetro; uma canaleta tem seção + largura + altura; um dissipador tem tipo + bacia; uma escada hidráulica tem número de degraus + altura de espelho. Forçar tudo numa tabela única vira um pesadelo de colunas nulas.

O padrão que recomendo (e que conversa direto com seu tooling GeoJSON/ARTESP) é núcleo comum + atributos específicos em JSONB validado por tipo:

Núcleo comum (toda drenagem tem): rodovia, km_ini/km_fim (ou km pontual), sentido, lado (BE/BD/canteiro/eixo), lat/long (entrada e saída nos lineares), categoria, tipo_dispositivo, material, extensao_m, estado_conservacao, data_inspecao, inspetor, fotos[], ncs[], obs.

Atributos específicos (JSONB, schema por tipo) — exemplos:

Tubular → diametro_mm, n_linhas, tem_ala, tem_dissipador
Canaleta → secao (triangular/trapezoidal/retangular/meia-cana), largura, altura, revestimento
Escada hidráulica → n_degraus, altura_espelho, largura, tem_dissipador_saida
Dissipador → tipo (bacia/enrocamento/blocos/escalonado), dimensoes
Taxonomia DNIT que eu sugiro embutir como enum/catálogo:

Superficial: valeta de proteção (corte/aterro), sarjeta (corte/aterro/canteiro), meio-fio, descida d'água (rápida / em degraus = escada hidráulica), saída/entrada d'água, dissipador de energia, caixa coletora/ligação, canaleta, valetão.

Profunda: dreno longitudinal, espinha-de-peixe, dreno sub-horizontal (DHP), dreno vertical, colchão drenante, dreno de pavimento.

Transversal (bueiros): BSTC/BDTC/BTTC (tubular), BSCC/BDCC (celular), com componentes ala, boca, berço, dissipador de saída.

Pronto — MVP rodável e testado. Você tem três peças que se conectam pelo catálogo:

Backend (backend/) — FastAPI + SQLModel. CRUD, /sync em lote com last-write-wins, /export/geojson e /catalogo. SQLite por padrão, troca pra Postgres só mudando DATABASE_URL.

PWA (pwa/) — app de campo offline-first. Formulário que se monta sozinho conforme o tipo (Ø pra tubular, nº de degraus pra escada hidráulica, seção pra canaleta), captura GPS com acurácia, foto com downscale, grava tudo em IndexedDB e sincroniza quando há rede. Instalável na tela inicial.

Catálogo (catalogo_dispositivos.py) — a fonte única. 15 tipos DNIT já mapeados nas três categorias. Pra adicionar um dispositivo novo, você edita só esse arquivo e roda python catalogo_dispositivos.py — o form do PWA se atualiza sozinho.

Pra subir agora: cd backend && pip install -r requirements.txt && uvicorn main:app --reload, e o PWA com python -m http.server (GPS/service worker exigem localhost ou HTTPS).
